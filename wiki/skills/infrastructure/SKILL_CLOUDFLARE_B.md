- Wrangler email setup or MCP email tools
- Keywords: email, cloudflare-email, routing, sending, transactional, deliverability

---

## Doctrine

Cloudflare Email Service = Email Sending (outbound) + Email Routing (inbound). Both run on Cloudflare's edge without needing a separate mail server.

---

## Quick Decision Tree

```
Need to SEND emails?
└─ YES → Email Sending API or Workers binding
   └─ Need templates? → Workers + React Email / MJML
   └─ Need bulk? → Batching + rate limits (max 1000/day free tier)

Need to RECEIVE emails?
└─ YES → Email Routing
   └─ Forward to Gmail? → Create route in Cloudflare dashboard
   └─ Process programmatically? → Workers Email Messages API
```

---

## Key Config

| Concern | Action |
|---------|--------|
| SPF | Add `v=spf1 include:_spf.mx.cloudflare.net ~all` |
| DKIM | Auto-generated by Cloudflare, no manual config |
| DMARC | Add `v=DMARC1; p=quarantine; rua=mailto:dmarc@domain.com` |
| Wrangler | `wrangler email bulk send --file emails.json` |
