## The Control Loop — Bounded Autonomous Updates

This is the core governance principle. **Autonomous does NOT mean agents freely mutate the canonical tree.**

### Four Artifacts Every Agent Emits After Non-Trivial Work

1. **Trace/event log** — what happened, step by step
2. **Evidence bundle** — raw outputs, command results, receipts
3. **Proposed wiki delta** — what should change in TREE777 and why
4. **Confidence/risk tag** — LOW / MEDIUM / HIGH / 888_HOLD

### The Promotion Ladder

| Evidence threshold | TREE777 destination | Who approves |
|-----------------|-------------------|-------------|
| Single event | `wiki/log.md` (append) | None — autonomous |
| Single failure | `wiki/scars/scar-*.md` (draft, status: proposed) | None — autonomous draft |
| Pattern 2–3× with evidence | `wiki/concepts/` or `wiki/skills/` (draft, status: proposed) | 888 JUDGE |
| Pattern across 3+ agents/platforms | Canonical skill | 888 JUDGE + Arif |
| Multi-step durable orchestration | `wiki/workflows/` | 888 JUDGE + Arif |
| Tree ontology / high-impact change | `intelligence-tree.md` or `SCHEMA.md` | **Arif only** |

### What Agents Can Do Autonomously (No Approval Required)

- Append to `wiki/log.md`
- Write raw evidence to `wiki/raw/notes/`
- Create scar draft with `status: proposed` in `wiki/scars/`
- Create skill/concept draft with `status: proposed` in `wiki/skills/` or `wiki/concepts/`
- Update existing stubs from `status: proposed` to `status: stub` (if filling in procedure)
- Fix wikilink typos in their own drafts

### What Requires 888 Deliberation (Agents Propose, Judge Decides)

- Promoting a proposed concept or skill to canonical
- Changing a canonical skill's `risk_band` from LOW to HIGH
- Creating a new workflow page
- Modifying any `floors:` field in a canonical skill
- Updating `evidence_required:` field on a canonical skill

### What Requires Arif Explicitly (888 HOLD → Sovereign)

- Changing the 7-layer ontology structure (`intelligence-tree.md`)
- Changing TREE777 architecture or adding new top-level directories
- Modifying `SCHEMA.md` governance rules
- Promoting a skill that affects 3+ agents federation-wide
- Any change to `TREE777.md` itself
