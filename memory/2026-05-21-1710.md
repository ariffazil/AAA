# arifOS MCP Kernel — Core Dependency Map & Agentic Architecture Analysis

> **Scope:** Five repositories — `arifos` (kernel), `AAA` (cockpit/surface), `wealth` (valuation organ), `geox` (earth intelligence), `A-FORGE` (runtime shell) — mapped for their canonical tool surfaces, runtime dependencies, cross-repo wiring, and missing/critical dependency gaps.

***

## Executive Summary

The arifOS ecosystem is a **constitutional multi-agent kernel** implementing the 13-floor governance doctrine across five repos, each playing a distinct role in the Architect–Auditor–Agent (AAA) trinity. All five repos share a common constitutional spine: `fastmcp` as the MCP transport primitive, `pydantic` for schema contracts, `asyncpg`/`supabase` for VAULT999 persistence, and `qdrant-client` for semantic memory. The key agentic dependency gaps lie in **transport coupling** (arifos assumes HTTP, A-FORGE assumes both stdio and HTTP), **LLM provider routing** (only A-FORGE has a proper multi-provider fallback chain), and **cross-repo semantic versioning** (arifos pins `fastmcp==3.2.4` while geox uses `fastmcp[tasks]>=0.7.0`, a silent version divergence risk).
