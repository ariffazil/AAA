# 2026-06-07 03:30Z — OpenCode 000♎ Revival (3-layer outage)

**TLDR:** 000♎ (arifOS_bot code-specialist persona) was returning HTTP 500 in AAA.
Three stacked failures. Fixed in 4 min. Now e2e green.

## What broke

1. **Stale `.init_session`** pointed to a session ID (`ses_15fef23...`) that was no
   longer in OpenCode's DB. Caused 404 "Session not found" on every message.

2. **Orphan manual `node opencode serve --port 4096`** was holding port 4096,
   so systemd `opencode.service` kept failing to bind (12 restart attempts, all
   "port 4096 in use"). Whoever started the orphan at 03:10 to "fix" the dead
   systemd service never killed it, so the orphan and systemd were fighting.

3. **Stale model config** in the init session — it was pinned to
   `provider=anthropic model=claude-sonnet-4`, which has no API key configured
   (opencode.jsonc is empty `{"$schema": ...}`). Cause of the original
   `ProviderModelNotFoundError` HTTP 500 even when the session existed.

## What fixed it

- Killed the orphan processes (PIDs 1996228, 1996254, 1993872, 1993880, 1993904).
- Restarted `systemctl start opencode.service` — bound 4096 cleanly.
- Created fresh init session `ses_15fdc3870ffenLHc4c7wFcW02X` (v3) using the
  default `big-pickle/opencode` bundled model (the only one with working creds).
- Updated `/root/.openclaw/workspace/bots/opencode-bot/.init_session` to v3.
- Patched `_read_init_session()` in `bot.py` to **liveness-check** the session
  before returning it. If the file points to a dead session, return None so
  `get_or_create_session()` will create a new one. Prevents future recurrence.
- Restarted `bot.py` (no systemd unit, plain `nohup`).

## Lessons (write down so future-me doesn't repeat)

- **Always check what else is on the port before assuming "service dead".**
  `ss -tlnp | grep <port>` first, then kill all holders, then start the
  canonical service. Orphan-vs-systemd fights on a port are silent killers.
- **`.init_session` is a foot-gun.** A dead session ID in that file causes
  silent 500/404 spam. Always validate before returning. (Now done.)
- **opencode.jsonc is empty.** No provider config beyond `$schema`. Only
  `provider=opencode model=big-pickle` works out of the box. If we want Claude
  or MiniMax via opencode, we have to add it to opencode.jsonc.
- **Bot has no systemd unit.** It runs as a backgrounded `nohup` process. If
  the server reboots, the bot dies silently. Add a systemd unit.
