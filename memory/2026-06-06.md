### 🪞 Critical meta-finding — TWO-COPY PROBLEM

**The editable install maps `arifosmcp` → `/opt/arifos/app/arifosmcp`** (per `__editable___arifos_2026_5_26_finder.py`), NOT to `/root/arifOS/`. So `arifos.service` actually reads `/opt/arifos/app/...`, not the dev copy in `/root/arifOS/`.

This means:
- Editing `/root/arifOS/...` is **invisible to live** (I lost ~30 min on this)
- The two copies drift unless someone keeps them in sync
- An `arifos` MCP restart from `/root/arifOS/` would still serve `/opt/arifos/app/...` (zombie code)

**F1/F2 violation potential:** a constitutional claim made against a fresh `/root/arifOS/` edit does NOT apply to the running kernel until the `/opt/arifos/app/` copy is also updated.

- (i) remove `arifos-core.pth` so the editable finder is the only resolver, OR
- (ii) make `/root/arifOS/` a symlink to `/opt/arifos/app/arifosmcp/`, OR
- (iii) move the editable install to point at `/root/arifOS/`
