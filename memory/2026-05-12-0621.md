### 6. Critical Finding — COHERENCE GAP

**Architecture is siloed. No automatic memory cross-pollination.**

- Hermes writes to: SQLite state.db
- OpenClaw writes to: flat files in /root/.openclaw/workspace/
- Both on same VPS — can read each other's files — but no automatic sync
- Current mitigation: both write to memory/YYYY-MM-DD.md when decision matters — but MANUAL, no enforcement

**Concrete risk:** A decision stored in Hermes SQLite may not be accessible to OpenClaw and vice versa. The 2026-05-11-1246.md Hermes self-correction was manually synced (example confirmed).

**Fix options (ranked):**
1. Shared memory bus (one SQLite both write to) ← best
2. Hermes reads OpenClaw MEMORY.md at boot ← easy
3. OpenClaw has Hermes SQLite query tool ← medium effort
