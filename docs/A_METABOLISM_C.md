## 3. WHAT EACH REFERENCE MEANS FOR arifOS

### 3.1 Official MCP — The Pipe

MCP is the standard connection layer between AI and external systems. It handles files, databases, search, tools, calculators, workflows.

**It does NOT provide:**
- Judgment
- Memory discipline
- Audit receipts
- Human veto
- Sovereign authority
- Tool trust ranking
- Constitutional gates

> MCP is the blood vessel, not the full organism.

**Relevance:** This is the base metabolism pipe. arifOS speaks MCP natively (Streamable HTTP + JSON-RPC, `2025-11-25`). Every organ exposes MCP. The pipe is standard; the governance is what's novel.

---

### 3.2 OpenAI Agents SDK + MCP — Closest Agent Body

Has tools, guardrails, orchestration, handoffs, human-in-the-loop, sessions, context management, MCP integration.

**What it does:**
- Agent has tools
- Agent can pass work to another agent
- Agent can keep session context
- Agent can use MCP
- Agent can pause for human approval

**What it does NOT have:**
- F13 sovereign veto
- VAULT999 immutable audit receipts
- GEOX/WEALTH/WELL domain organs
- SEAL/HOLD/VOID verdict logic
- Constitutional floor enforcement (F1-F13)
- Hazard band classification (GREEN→BLACK)
- Scar metabolization (learn from failures as permanent constraints)

> Closest public agent framework. Still missing the constitution.

---

### 3.3 Code2MCP — The Forge Pattern

Takes GitHub repositories, understands them, builds MCP servers from them, runs them, checks for failure, fixes the failure. A "run, review, fix" loop.

**Maps to:** A-FORGE tool-organ factory. The pattern of read→understand→build→test→repair is exactly what A-FORGE does when synthesizing tools.

**Difference:** Code2MCP has no constitutional gate before execution. arifOS requires: classify → preflight → SEAL → bounded execute → VAULT999 record.

---

### 3.4 ScaleMCP — Tool Memory

Helps agents dynamically find MCP tools, stores tool knowledge, synchronizes with MCP servers, avoids fixed manual tool lists.

**Maps to:** The tool metabolism question. Not just "do I have tools?" but:
- Does the agent know which tool matters now?
- Does it remember tool behavior?
- Does it avoid calling the wrong tool?
- Does the tool list stay clean?

**Difference:** arifOS has a tool registry with capability tags and hazard bands. ScaleMCP has dynamic discovery without constitutional classification.

---

### 3.5 MCPShield — Immune System

Checks tools before and after use. Watches what tools do. Updates trust after use. Prevents agents from wrongly trusting bad MCP servers.

**Maps to:** arifOS immune system — HARAM scan, SCAR metabolization, F12 injection defense.

```
MCPShield  = technical immune layer (check tool behavior)
arifOS     = constitutional immune layer (check tool authority)
```

Both are needed. MCPShield catches malicious tools. arifOS catches unauthorized use of legitimate tools.

A 2026 paper on deploying MCP agents identifies exactly where production systems struggle: identity propagation, error clarity, observability, tool budgeting, stable audit, safe execution.

**This is the AMBER gap.** Your conformance check showed strengths in identity, permission gate, vault trace, session start, and mutation hold — but protocol version reporting, schema echo stability, and WELL cooling were missing.

The missing work matches exactly where production MCP systems struggle.

### 3.7 NotebookLM + Claude MCP — Research Metabolism

Connects a research notebook to a reasoning model via MCP. The model can reason over stored documents as a personal research hub.

**Maps to:** A-ARCHIVE + reasoning model. Small-scale document memory + reasoning.
