## 0. The Breakthrough

> **An agent becomes trustworthy not when it is more intelligent, but when every movement from thought to consequence is bound to identity, authority, evidence, memory, execution and receipt.**

Most agent systems treat intelligence as the centre:

```
prompt → model → tool → result
```

arifOS after this doctrine becomes:

```
human intent
    ↓
authenticated identity
    ↓
bounded authority
    ↓
evidence-aware reasoning
    ↓
classified consequence
    ↓
controlled execution
    ↓
verified result
    ↓
memory revision
    ↓
immutable receipt
    ↓
cooling and learning
```

The agent is no longer merely a model that can call tools. It becomes a **governed living process** — not biologically alive, not conscious by declaration, but operationally alive: able to recognise who it serves, know its limits, act within a bounded world, remember, distinguish fact from inference, record consequences, detect recurring failures, improve without silently taking power, and **stop when truth or authority is missing**.

## 4. The 11-Step Chain

The permanent engineering principle:

> Classify first. Authorise second. Act third. Verify fourth. Remember fifth. Seal last. Cool always returns through governance.

```
1.  Understand the request
2.  Resolve the actor
3.  Classify the proposed action
4.  Calculate required authority
5.  Verify available authority
6.  Check evidence and consequences
7.  Issue a narrow capability
8.  Execute the exact action
9.  Verify the actual result
10. Update memory
11. Write the correct receipt
12. Cool the session and learn (through governance)
```

**Anti-pattern (before EUREKA):**
```
call tool → encounter gate → patch gate → encounter another gate → bypass gate
```

**Correct flow (after EUREKA):**
Each gate receives the facts it needs before enforcement begins.
