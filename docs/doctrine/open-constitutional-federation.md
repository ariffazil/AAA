## 8. Five Forms of Openness

A GitHub repository is not constitutional openness. A genuinely open substrate requires five forms of openness.

```
O_c = O_t × O_e × O_g × O_i × O_s
```

Where:
- **O_t** = Technical openness — runtime, schemas, capability graph, enforcement paths can be inspected
- **O_e** = Epistemic openness — claims preserve evidence, uncertainty, dissent and provenance
- **O_g** = Constitutional openness — rules governing authority and irreversible action are explicit
- **O_i** = Institutional openness — no single model, agent or organisation is the sole witness and judge
- **O_s** = Sovereign openness — affected human can challenge, revoke, correct and veto

If any critical dimension approaches zero, constitutional openness collapses. Publishing source code while retaining hidden authority and unchallengeable execution does not solve the problem.

## 9. Federation Is Not a Swarm

Federation requires meaningful independence.

**Fake federation:** same provider, same model family, same system policy, same memory, same incentives, same controller. This is distributed execution under centralized sovereignty.

**Real federation:** different evidence organs, different models or methods, independent witnesses, explicit authority boundaries, preserved disagreement, shared constitutional protocol, human sovereign veto.

A valid federation must allow an organ to say:

- "I do not have the evidence."
- "My result conflicts with GEOX."
- "WEALTH's optimization harms WELL."
- "The model changed and continuity is uncertain."
- "The proposed action exceeds authority."
- "The majority is wrong."
- "F13 must decide."

Federation is not agreement. Federation is the capacity to preserve legitimate disagreement without allowing the system to fragment or conceal conflict.
