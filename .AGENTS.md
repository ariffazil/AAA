## What is AAA?

AAA is the **Control Plane** and **Identity Authority** for the arifOS Federation:

- **Identity registry** — Sovereign identity binding (`ARIF.md`, `IDENTITY.md`, `SOUL.md`)
- **React UI** — Constitutional operator dashboard + AI chat interface
- **A2A Gateway** — Agent-to-Agent mesh protocol + federation routing
- **State initialization** — Seed governance contracts and operator context

---

## Structure

```
AAA/
├── src/
│   ├── main.tsx           ← React entry point
│   ├── App.tsx            ← Root component
│   ├── Cockpit.tsx        ← Operator dashboard
│   ├── ai/                ← Chat UI + MCP integration
│   ├── gateway/           ← A2A v1.0.0 TypeScript server
│   ├── adapter/           ← GovernanceAdapter (routes intent to A-FORGE)
│   ├── seed/              ← Identity docs (SOUL.md, IDENTITY.md)
│   ├── components/ui/     ← Radix + Tailwind components
│   └── hooks/             ← React hooks
│
├── a2a-server/            ← Standalone Express A2A gateway (port 3001)
├── contracts/             ← YAML governance contracts
├── package.json           ← Node.js manifest
└── vite.config.ts         ← Vite build config
```

## Quick Start

```bash
# Install
npm install

# Dev server
npm run dev                # Vite dev server (http://localhost:5173)

# Build
npm run build

# A2A validation
npm run validate:aaa
npm run a2a:server        # Start A2A gateway
```

---

## Key Entrypoints for Agents

- **React App:** `npm run dev` — Cockpit dashboard, AI chat
- **A2A Gateway:** `npm run a2a:server` — Agent mesh protocol
- **Health:** `http://localhost:3001/health` or `https://aaa.arif-fazil.com/health`
