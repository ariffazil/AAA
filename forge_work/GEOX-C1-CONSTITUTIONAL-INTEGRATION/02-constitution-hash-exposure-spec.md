## Problem

GEOX responses include `constitution_hash: "unknown"`. GEOX has internal hashes (`registry_hash: reg-hash-35d798a`, `geox_version: v2026.06.05`, `contract_epoch: 2026-06-05-GEOX-35TOOLS-v2.0`) but does not expose a constitutional anchor that ties to a known external canon.

This breaks F10 (Ontology) cross-validation. Any consumer (arifOS, A-FORGE, human reviewer) cannot verify that the GEOX response was generated by the same doctrine that was sealed.

**Live receipt (2026-06-06T14:06Z):**
```json
{
  "constitution_hash": "unknown",
  "registry_hash": "reg-hash-35d798a",
  "contract_epoch": "2026-06-05-GEOX-35TOOLS-v2.0",
  "geox_version": "v2026.06.05"
}
```

## Goal

Expose a stable, verifiable `constitution_hash` that:
1. Anchors the response to the **sealed GEOX doctrine** (currently `GEOX-CANON-31.md` at `forge_work/GEOX-CANON-31.md`).
2. Changes only when the doctrine is re-sealed.
3. Is verifiable by any consumer with read access to the doctrine file.

## Design

### Two options

| Option | Compute time | Cost | Pros | Cons |
|---|---|---|---|---|
| (a) **Frozen at deploy** | Once at install | ~1ms per call (lookup) | Fast, deterministic, no I/O on hot path | Hash stale if doctrine edited without re-deploy |
| (b) **Live digest** | Every call | ~5-10ms per call (file I/O + sha256) | Always fresh | Slow, I/O on hot path, race condition if file edited mid-call |

**Recommendation:** (a) Frozen.

Reasoning: doctrines are sealed, not edited. If the doctrine changes, the deploy changes. The hash should not change silently. This is the same discipline as `arifos://doctrine` in the arifOS substrate.

### Frozen hash implementation

**Deploy step:**
```bash
# At install/upgrade time
sha256sum /opt/geox/canon/GEOX-CANON-31.md | cut -d' ' -f1 > /opt/geox/canon/.constitution_hash
# Embed in environment for the GEOX server to read
export GEOX_CONSTITUTION_HASH=$(cat /opt/geox/canon/.constitution_hash)
```

**Response envelope (modified):**
```json
{
  "constitution_hash": "sha256:861babd1...",
  "constitution_source": "forge_work/GEOX-CANON-31.md",
  "constitution_sealed_at": "2026-06-06T13:50Z"
}
```

### New fields

| Field | Type | Source | Required? |
|---|---|---|---|
| `constitution_hash` | string (sha256) | `GEOX_CONSTITUTION_HASH` env var | YES — must NOT be `"unknown"` |
| `constitution_source` | string (path) | `GEOX_CONSTITUTION_SOURCE` env var | YES — points to the doctrine file |
| `constitution_sealed_at` | string (ISO 8601) | `GEOX_CONSTITUTION_SEALED_AT` env var | YES — when the doctrine was sealed |

### Failure modes

| Condition | Behavior |
|---|---|
| `GEOX_CONSTITUTION_HASH` env var not set | `888_HOLD: CONSTITUTION_NOT_ANCHORED` |
| Env var set but doctrine file missing | `888_HOLD: CONSTITUTION_FILE_MISSING` |
| Env var hash doesn't match sha256 of file | `888_HOLD: CONSTITUTION_HASH_MISMATCH` |
| Env var set, file present, hash matches | Return `constitution_hash` in response |

**Do not fall back to `"unknown"`.** This is a sealed invariant.

## Implementation sketch

**File:** `geox/canon/constitution.py` (new, ~40 lines)

```python
import hashlib
import os

def load_constitution():
    """Load and verify constitution hash at startup. Fails closed."""
    path = os.environ.get("GEOX_CONSTITUTION_SOURCE")
    expected_hash = os.environ.get("GEOX_CONSTITUTION_HASH")
    sealed_at = os.environ.get("GEOX_CONSTITUTION_SEALED_AT")
    if not path or not expected_hash or not sealed_at:
        raise ConstitutionNotAnchored("888_HOLD: CONSTITUTION_NOT_ANCHORED")
    if not os.path.exists(path):
        raise ConstitutionNotAnchored("888_HOLD: CONSTITUTION_FILE_MISSING")
    actual_hash = "sha256:" + hashlib.sha256(open(path, "rb").read()).hexdigest()
    if actual_hash != expected_hash:
        raise ConstitutionNotAnchored("888_HOLD: CONSTITUTION_HASH_MISMATCH")
    return {
        "constitution_hash": expected_hash,
        "constitution_source": path,
        "constitution_sealed_at": sealed_at,
    }
```

**Wire-up:** `geox/server.py` (modify, ~5 lines) — call `load_constitution()` at startup, inject into response envelope.

## Acceptance criteria

1. **W0 self-report:** `geox_system_registry_status` returns `constitution_hash != "unknown"`.
2. **W1 raw probe:** Direct JSON-RPC response includes `constitution_hash`, `constitution_source`, `constitution_sealed_at`.
3. **W2 connector probe:** Same — the connector does not strip these fields.
4. **Verifiability:** A consumer with read access to `forge_work/GEOX-CANON-31.md` can compute the same hash and match.
5. **Failure mode:** If `GEOX_CONSTITUTION_HASH` is unset, `geox_system_registry_status` returns `888_HOLD: CONSTITUTION_NOT_ANCHORED`.

## Reversibility

✅ All changes are in GEOX substrate.
- Rollback: unset env vars, restart GEOX.
- Federation contract change: response envelope gains 3 new fields (additive, backward compatible if consumers ignore unknown fields — same discipline as arifOS strict-schema).

## Sunset policy

This sub-forge modifies the GEOX substrate. It is **not** part of the tactical-bridge sunset. Once shipped, it is part of the canon.

## Out of scope

- Does not compute `constitution_hash` live. Frozen at deploy (see design).
- Does not track multiple historical hashes. One frozen hash per deploy.
- Does not federate the hash. Each organ (arifOS, GEOX, WEALTH, WELL) has its own hash.
